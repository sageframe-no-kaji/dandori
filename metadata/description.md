# Dandori — description

A discipline for preparing bounded specifications that an autonomous coding agent can execute as one unit of work — published as a Claude Skill.

The current generation of coding agents can run unattended for hours on tasks they will get spectacularly wrong if briefed casually. The failure is structural: a prompt is a sentence, and a sentence is the wrong artifact for autonomous work. Dandori is the answer in methodology form — a discipline for converting planning conversations into bounded, reviewable specifications the agent treats as one unit of work. The name is exact: a Japanese workshop's dandori (段取り) is the preparation that makes the skilled labor possible. The deeper argument is about authorship of agent work — who decided what gets built, and where in the artifact chain that decision is recorded; without Dandori the answer is nobody, nowhere, and with it the answer is the practitioner, in the spec. That distinction is the difference between agent work that is auditable and agent work that is autopilot; it sits alongside the Ho System, where Ho structures the practitioner's session and Dandori structures the agent's brief.

Dandori is a methodology published as a Claude Skill — a structured markdown instruction set an agent reads to author a task spec — distributed via GitHub.
